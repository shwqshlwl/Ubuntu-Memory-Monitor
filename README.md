# Ubuntu 内存监控器 (Ubuntu Memory Monitor)

![License](https://img.shields.io/badge/license-MIT-blue.svg)
![Python](https://img.shields.io/badge/python-3.6%2B-green.svg)
![Platform](https://img.shields.io/badge/platform-Ubuntu%20%7C%20Linux-lightgrey.svg)

一个专为 Ubuntu/Linux 桌面环境打造的**轻量级内存监控与自动干预工具**。

当您的 Ubuntu 系统经常因为跑满内存（如开启大量 IDE、Docker 容器、Elasticsearch 等）而突然卡死、甚至连终端都无法呼出时，这个小工具可以作为您系统的"最后一道防线"，在崩溃前通过**全屏醒目弹窗**和**阶梯式自动执行系统指令**实现"断臂求生"。

---

## ✨ 核心特性 (Features)

- 🪶 **极度轻量**：基于 Python 和内置的 `tkinter` 构建，无需安装 Electron 或笨重的 GUI 框架，运行时几乎不占用系统资源。
- 🚨 **全屏醒目警告**：内存达到警告线时，会在屏幕中央弹出**无边框全色块警告窗口**（橙色/红色），带闪烁边框动画，强制吸引注意力。
- 🪜 **阶梯式干预策略**：支持配置多条急救指令（例如先关 Chrome，不行再停 Elasticsearch），系统会按优先级逐条执行，**每条命令间隔 15 秒**，直到内存恢复安全水平。
- 🔐 **安全的 Sudo 提权**：干预指令中常包含 `sudo`，本工具支持在界面中临时输入 Sudo 密码，密码**仅保存在内存中，绝不写入硬盘配置文件**，兼顾自动化与安全性。
- ⚙️ **智能进程管理**：内置可视化的进程管理器，实时按内存占用排序，自动识别 Java/Python/Node 等包装型进程并显示命令行参数，鼠标悬停可查看完整进程名；支持一键结束，遇高权限进程自动使用暂存的 Sudo 密码提权强杀。
- 📊 **可视化日志看板**：直观展示实时内存进度、当前正在执行的干预指令以及执行成功/失败的反馈日志，同时自动写入 `monitor.log` 文件便于事后排查。
- 🔒 **单实例守护**：同一时间只能运行一个监控实例，防止重复启动冲突。
- 🪟 **智能窗口管理**：监控运行中关闭窗口会自动最小化到任务栏后台运行，不会误关；暂停后可安全退出。

## 🚀 快速开始 (Quick Start)

### 1. 获取代码
```bash
git clone https://github.com/您的用户名/Ubuntu-Memory-Monitor.git
cd Ubuntu-Memory-Monitor
```

### 2. 一键安装依赖
项目内提供了一个初始化脚本，会自动帮您安装系统级的 `python3-tk` 以及创建虚拟环境并安装 `psutil`：
```bash
chmod +x setup.sh
./setup.sh
```
*(注：由于网络原因，如果 apt 更新卡在第三方源，脚本自带了 5 秒超时跳过机制。)*

### 3. 创建桌面快捷方式 (可选)
如果您希望在系统的"应用程序列表"或桌面上直接点击图标启动：
```bash
chmod +x setup_desktop.sh
./setup_desktop.sh
```

### 4. 启动监控器
您可以直接从应用抽屉搜索"内存监控"点击打开，或者在终端中运行：
```bash
./venv/bin/python gui.py
```

## 🛠️ 使用说明

启动界面后，您需要配置以下几个关键参数：

1. **警告阈值 (%)**：达到此百分比时（如 `80%`），弹出橙色醒目警告，提示您手动清理内存。
2. **危急阈值 (%)**：达到此百分比时（如 `90%`），不仅会弹出红色闪烁警告，还会**自动触发**下方的干预指令。
3. **干预指令**：
   - 支持多行输入，**每行代表一条独立的命令**。
   - **优先级从上到下**。第一条执行完若内存仍未降下，则等待 15 秒后执行第二条，以此类推。
   - 所有指令执行完毕后若内存仍危急，会弹窗提示手动干预。
   - 示例：
     ```bash
     sudo systemctl stop elasticsearch
     killall chrome
     sudo systemctl restart docker
     ```
4. **Sudo 密码 (可选)**：如果您的干预指令中包含了 `sudo` 开头的命令，请在此处填入您的系统密码。程序会自动通过标准输入流（stdin）提权。

配置完成后，点击 **"保存配置"**，然后点击右下角的 **"▶ 启动监控"** 即可让其在后台守护您的系统。

### 进程管理标签页

- 点击 **"🔄 刷新列表"** 获取当前按内存排序的进程
- Java/Python/Node 等进程会显示命令行参数（如 `java [/usr/share/elasticsearch/...]`）
- 鼠标悬停在进程名上可查看完整名称
- 选中进程后点击 **"❌ 结束选中进程"** 即可终止
- 遇到需要 root 权限的进程，会自动读取 Sudo 密码提权

### 窗口行为

- 监控运行中点击右上角 **❌**：窗口**最小化到任务栏**，监控继续运行
- 监控暂停后点击右上角 **❌**：弹出确认框，确认后退出
- 如果已经有一个实例在运行，再次启动会提示"已在运行中"并自动退出

## ⌨️ 命令行模式 (monitor.py)

如果您不需要 GUI，也可以纯终端运行：

```bash
./venv/bin/python monitor.py --sudo-pwd 你的密码
```

特点：
- 纯终端输出，不依赖 GUI
- `--sudo-pwd` 参数传入密码以支持 sudo 命令
- 同样支持阶梯式干预、15 秒超时、单实例锁
- Ctrl+C 安全退出

## ❓ 常见问题 (FAQ)

**Q1: 为什么点击"生成桌面快捷方式"后，桌面上没有出现图标？**  
A: 部分 Ubuntu 系统的 `~/桌面` 目录可能因挂载问题变为"只读"，导致无法写入。不用慌，脚本已成功将应用注册到 `~/.local/share/applications`。按下 `Super(Win)` 键搜索"内存监控"即可找到。

**Q2: 为什么我的干预指令执行失败，日志提示"执行超时"？**  
A: 每条指令默认有 15 秒的执行时间限制，防止某条僵死命令阻塞整个干预流程。如果是 `sudo` 命令超时，请检查是否在界面上正确输入了 Sudo 密码。

**Q3: 内存降下来后，干预流程会重置吗？**  
A: 会的。当物理内存占用下降到 **(危急阈值 - 5%)** 的安全水位以下时，程序会自动将干预进度归零。您也可以随时点击界面上的"↺ 重置干预状态"按钮手动归零。内存再次超过阈值时会重新触发弹窗和干预。

**Q4: 如何在桌面快捷方式下排查崩溃问题？**  
A: GUI 启动后所有日志自动写入 `monitor.log` 文件（与 `gui.py` 同目录），可在终端中 `tail -f monitor.log` 实时查看。

**Q5: 进程列表里找不到 Elasticsearch？**  
A: Elasticsearch 运行在 JVM 上，进程名显示为 `java`。本工具会附加命令行参数，如 `java [/usr/share/elasticsearch/...]`，鼠标悬停可看完整路径。
